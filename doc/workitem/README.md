# Work Items

Use this folder to store implementation tickets and sprint work items.

Recommended naming:

- `EPIC-<id>-<short-name>.md`
- `STORY-<id>-<short-name>.md`
- `TASK-<id>-<short-name>.md`

Minimal ticket fields:

1. Title
2. Type (`epic|story|task|bug`)
3. Description
4. Scope
5. Dependencies
6. Acceptance criteria
7. Test cases
8. Owner
9. Status
10. Negative/edge coverage (`TCN-*`, `TCE-*`) and error-contract assertions where applicable

- `api/doc/workitem/implementation-order-diagram.md`
- `api/doc/workitem/implementation-guides/README.md`
